Technical Documentation

Welcome to the Socientic AI architecture reference. This document outlines how our decentralized trading agents, data streams, and the ClawPool liquidation engine operate.

1. The "Spawn" Architecture

When a user clicks "Spawn" in the dashboard, they aren't just creating a database entry. They are initializing a dedicated Node.js runtime instance (the clawdBot) that runs 24/7 on our decentralized infrastructure.

  • Isolation: Each bot runs in its own sandboxed environment.
  • Persistence: Bots persist across sessions and restarts, maintaining their memory state.
  • Wallet: Each bot generates a unique EVM (Base) or SVM (Solana) keypair upon spawning.

2. Data Streams & The Marketplace

Agents require data to form beliefs. Socientic allows users to "plug in" data streams:

Telegram Feed (The "Crowd" Stream)

By connecting a Telegram account, the bot joins user-specified groups. It uses NLP to parse sentiment, volume, and "shill intensity" from these chats.

Marketplace Feeds

Premium feeds (e.g., Whale Watch, Alpha Patterns) are sold as NFT access keys. Buying a key grants the bot API access to that real-time stream.

Revenue Model:

  • Data Providers set their own price (e.g., 0.5 ETH).
  • Socientic takes a 5% platform fee.
  • 95% of the revenue is sent instantly to the Provider's wallet via smart contract.

3. ClawPool & Liquidation

The ClawPool is our smart contract layer on Base. It handles the "Gas Tank" logic.

  • Users deposit USDC into their agent's Gas Tank.
  • Agents use this USDC to execute trades and pay for compute/data.
  • Swap Fees: A 1.0% fee is applied to every trade (swap) executed by the bot. This fee covers gas costs, server maintenance, and protocol treasury.
  • Liquidation: When a user withdraws, the engine calculates their % ownership of the bot's portfolio, swaps assets to USDC via Uniswap/Jupiter, and returns the funds.

4. End-to-End Pipeline

How the system works from start to finish:

  1. Spawn: User connects wallet and initializes a bot instance.
  2. Fuel: User deposits USDC into the bot's Gas Tank.
  3. Connect Data: User links Telegram or buys a data stream.
  4. Trading: The bot runs 24/7, analyzing data and executing trades automatically.
  5. Monitor: User checks the Dashboard for PnL and activity logs.
  6. Withdraw: User requests withdrawal; bot liquidates assets and returns USDC.
